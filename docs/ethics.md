# 伦理与安全边界

故影处理的是人们对逝者、亲友、老师、同学和社区成员的记忆。这个主题天然带有情感、隐私和身份风险,因此伦理边界必须写进产品和代码。

## 基本声明

本项目生成的是**基于记忆的 AI 纪念肖像**,不是历史照片,也不是身份识别结果。

所有导出图片和公开展示都应明确标注:

```text
AI-generated memory portrait. Not an original photograph.
AI 生成的记忆肖像,非真实历史照片。
```

## 禁止用途

本项目不应用于:

- 身份识别、人脸搜索或监控。
- 执法、司法、取证或新闻证据生成。
- 冒充真实照片、制造虚假历史材料。
- 未经授权公开逝者、家属或相关人物的敏感资料。
- 生成用于欺骗、骚扰、羞辱或商业冒用的图像。

## 产品约束

建议在产品层面内置以下约束:

- 不提供真实人脸数据库检索。
- 不提供“找到相似真人”的功能。
- 默认私有保存,公开分享需要主动确认。
- 导出图必须带 AI 生成标识或元数据。
- 人物档案应记录记忆来源和确定度。
- 示例数据必须使用虚构人物。

## 不确定性标注

每个样貌字段都应支持确定度:

```yaml
certainty: high | medium | low | unknown
```

示例:

```yaml
eyes:
  value: "眼睛不大,眼神温和"
  certainty: "medium"
  sources: ["学生A", "学生B"]
```

## 多人记忆差异

当多位贡献者记忆不一致时,系统不应强行合并成一个“唯一事实”。更好的做法是保留差异:

```text
女儿记得: 年轻时脸偏长,颧骨明显。
学生记得: 晚年脸更圆,笑起来很慈祥。
邻居记得: 常戴深色头巾。
```

这些差异可以生成不同版本的纪念肖像,并在导出档案中说明。

## 公开分享建议

公开分享前应确认:

- 分享者是否有合适的家庭或社区授权。
- 是否移除了不必要的个人敏感信息。
- 是否明确说明图像是 AI 生成。
- 是否尊重当地文化、宗教和家庭习俗。
