# 产品愿景

故影 Memory Portrait 的核心问题不是“AI 能不能画出一张脸”,而是:

> 当一个人没有留下照片时,我们能否以尊重、诚实、可追溯的方式保存大家共同记得的样子?

## 用户场景

- 家人想为没有留下照片的长辈整理纪念资料。
- 学生想回忆小学老师、同学或邻里长辈的样子。
- 村史、校史、地方志项目希望保存口述历史中的人物记忆。
- 多位亲友对同一位故人的记忆不完全一致,希望把差异也保存下来。

## 设计原则

### 记忆优先

最终产物不只是图片,还包括记忆来源、描述文本、生成历史和不确定性标注。

### 选择优先于描述

很多人无法准确描述鼻梁、眼距、嘴型等参数,但可以判断“这一张更像一点”。因此核心交互采用候选图选择和逐轮收敛。

### 不确定性可见

系统应显示哪些信息来自明确记忆,哪些来自推测或模型生成。模糊之处不应被伪装成确定事实。

### 私密默认

人物档案默认只在本地或私有环境中保存。公开分享必须由用户主动选择,并附带 AI 生成说明。

## 成功标准

第一阶段的成功不是生成完美肖像,而是验证:

- 用户能否完成一次人物记忆档案创建。
- 多轮选择是否让候选图更接近用户主观记忆。
- 导出的档案是否清楚表达“这是基于记忆的纪念肖像,不是历史照片”。
- 没有 GPU 的贡献者是否也能运行和改进项目。
